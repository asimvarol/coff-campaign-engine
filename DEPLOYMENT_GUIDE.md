# 🚀 Deployment Guide - Coff Campaign Engine

## Quick Deploy (Vercel - Recommended)

### 1. First Time Setup

```bash
# Login to Vercel (if not already logged in)
vercel login

# Link project to Vercel
cd /Users/dev/coff-campaign-engine
vercel link
```

You'll be asked:
- **Set up and deploy?** → Yes
- **Which scope?** → Your account/team
- **Link to existing project?** → No (if first time)
- **What's your project's name?** → coff-campaign-engine
- **In which directory is your code located?** → ./

### 2. Deploy to Staging (Preview)

```bash
# Deploy current branch to preview URL
vercel

# Wait for deployment...
# You'll get a URL like: https://coff-campaign-engine-xyz.vercel.app
```

**Preview deploys are automatic for:**
- Every git push to non-main branches
- Pull requests

### 3. Add Environment Variables

**Via Vercel Dashboard:**
1. Go to https://vercel.com/dashboard
2. Select your project
3. Settings → Environment Variables
4. Add these (copy from `.env`):

```bash
# Required for Production
DATABASE_URL=postgresql://user:pass@host:5432/db_name
NEXTAUTH_SECRET=<generate-new-secret>
NEXTAUTH_URL=https://your-domain.vercel.app

# OAuth (Optional but recommended)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# AI Services
FAL_AI_KEY=8032b3eb-a441-42ab-9ab0-1578262d8fbc:ed5717a6db5d5c8983c009f1fa5cf8de
OPENAI_API_KEY=sk-...

# Feature Flags
ENABLE_AUTOPILOT=true
ENABLE_AGENCY_MODE=true
```

**Generate new NEXTAUTH_SECRET:**
```bash
openssl rand -base64 32
```

**Via CLI (faster):**
```bash
# Add environment variables
vercel env add DATABASE_URL production
vercel env add NEXTAUTH_SECRET production
vercel env add NEXTAUTH_URL production
# ... repeat for all vars
```

### 4. Deploy to Production

```bash
# Deploy main branch to production
vercel --prod

# Or merge to main and auto-deploy (if configured)
git checkout main
git merge feat/analytics-agency
git push origin main
```

### 5. Setup Custom Domain (Optional)

```bash
# Add custom domain
vercel domains add yourdomain.com

# Vercel will give you DNS settings:
# - A record: 76.76.21.21
# - CNAME: cname.vercel-dns.com

# Wait for DNS propagation (~5 minutes)
# SSL certificate auto-generated by Vercel
```

### 6. Post-Deployment Checks

```bash
# Check deployment logs
vercel logs https://your-deployment-url.vercel.app

# Test production build locally first
bun run build
bun start

# Run Lighthouse audit on production
lighthouse https://your-domain.vercel.app --view
```

## Opsiyon 2: Docker (Self-Hosted)

### 1. Build Docker Image

```bash
# Build production image
docker build -t coff-campaign-engine:latest .

# Or use docker-compose
docker-compose -f docker-compose.prod.yml build
```

### 2. Run Production Stack

```bash
# Start all services (PostgreSQL + Redis + Web)
docker-compose -f docker-compose.prod.yml up -d

# Check logs
docker-compose -f docker-compose.prod.yml logs -f web

# Visit http://localhost:3000
```

### 3. Environment Variables

Create `.env.production`:
```bash
NODE_ENV=production
DATABASE_URL=postgresql://postgres:postgres@postgres:5432/coff_campaign_engine
REDIS_URL=redis://redis:6379
NEXTAUTH_URL=https://yourdomain.com
NEXTAUTH_SECRET=<your-secret>
# ... rest of vars
```

### 4. Reverse Proxy (Nginx)

```nginx
# /etc/nginx/sites-available/coff-campaign-engine
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Enable SSL with Certbot:
```bash
sudo certbot --nginx -d yourdomain.com
```

## Opsiyon 3: VPS (Manual Deploy)

### 1. SSH into Server

```bash
ssh user@your-server.com
```

### 2. Install Dependencies

```bash
# Install Bun
curl -fsSL https://bun.sh/install | bash

# Install Node.js (if needed)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install PostgreSQL & Redis
sudo apt-get install postgresql redis-server
```

### 3. Clone & Build

```bash
# Clone repository
git clone https://github.com/asimvarol/coff-campaign-engine.git
cd coff-campaign-engine

# Checkout production branch
git checkout main

# Install dependencies
bun install

# Setup environment
cp .env.example .env.production
nano .env.production  # Edit with production values

# Run database migrations
cd packages/db
DATABASE_URL="..." npx prisma migrate deploy

# Build production bundle
cd ../..
bun run build
```

### 4. Process Manager (PM2)

```bash
# Install PM2
npm install -g pm2

# Start application
pm2 start npm --name "coff-web" -- start
pm2 startup
pm2 save

# Monitor
pm2 logs coff-web
pm2 monit
```

## Database Setup (Production)

### Option A: Vercel Postgres (Easiest)

```bash
# Install Vercel Postgres
vercel postgres create coff-campaign-engine-db

# Get connection string
vercel postgres connect

# Add to environment variables
vercel env add DATABASE_URL production
```

### Option B: External Provider

Popular options:
- **Supabase** (PostgreSQL): https://supabase.com
- **PlanetScale** (MySQL): https://planetscale.com
- **Railway**: https://railway.app
- **Neon** (Serverless Postgres): https://neon.tech

Steps:
1. Create database instance
2. Copy connection string
3. Add to Vercel environment variables
4. Run migrations:
   ```bash
   DATABASE_URL="..." npx prisma migrate deploy
   ```

### Option C: Self-Hosted

```bash
# PostgreSQL on VPS
sudo -u postgres createdb coff_campaign_engine
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'secure_password';"

# Redis
sudo systemctl start redis-server
sudo systemctl enable redis-server
```

## CI/CD Pipeline (GitHub Actions)

Already configured in `.github/workflows/ci.yml`:

```yaml
# Automatic deployment on push to main
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to Vercel
        env:
          VERCEL_TOKEN: ${{ secrets.VERCEL_TOKEN }}
        run: vercel deploy --prod --token=$VERCEL_TOKEN
```

**Setup:**
1. Generate Vercel token: https://vercel.com/account/tokens
2. Add to GitHub Secrets: `VERCEL_TOKEN`
3. Push to main → auto-deploy
